# Binary Asset Embedder

Converts binary files into base64-encoded strings suitable for embedding
directly in source code. This is a standard technique for:

- Single-file HTML documents that embed images or fonts.
- WASM modules embedded in JavaScript bundles.
- Configuration files that reference assets without a separate asset server.
- Test fixtures that need to ship a binary alongside the test code.

## Why Base64?

Binary files contain byte values that cannot be directly embedded in text
formats (JSON, YAML, Python string literals, etc.) without corruption.
Base64 encoding converts any binary data into a safe ASCII representation.
It increases the file size by ~33% but is universally supported and widely
used for exactly this purpose (data URIs, JWT tokens, email attachments).

## Usage

### Encode a single file

```python
import base64, pathlib

def encode_asset(path: str) -> str:
    """Return the base64-encoded content of a binary file."""
    raw = pathlib.Path(path).read_bytes()
    return base64.b64encode(raw).decode("ascii")

# Example: embed a PNG icon
icon_b64 = encode_asset("assets/icon.png")
print(f'ICON_DATA = "{icon_b64}"')
```

### Embed as a data URI (for HTML/CSS)

```python
import base64, mimetypes, pathlib

def to_data_uri(path: str) -> str:
    """Return a data URI for a binary file."""
    mime, _ = mimetypes.guess_type(path)
    mime = mime or "application/octet-stream"
    raw = pathlib.Path(path).read_bytes()
    encoded = base64.b64encode(raw).decode("ascii")
    return f"data:{mime};base64,{encoded}"

# Example: inline a CSS font
font_uri = to_data_uri("fonts/MyFont.woff2")
css = f"@font-face {{ src: url('{font_uri}'); }}"
```

### Decode back to binary

```python
import base64, pathlib

def decode_asset(b64_string: str, output_path: str) -> None:
    """Decode a base64 string and write the result to a file."""
    raw = base64.b64decode(b64_string)
    pathlib.Path(output_path).write_bytes(raw)
```

## Batch Encoding

```
agent run binary-asset-embedder --source assets/ --output src/assets_generated.py
```

Generates a Python module with one constant per asset:

```python
# Auto-generated by binary-asset-embedder — do not edit manually
ICON_PNG = "iVBORw0KGgoAAAANSUhEUg..."
LOGO_SVG_B64 = "PHN2ZyB4bWxucz0iaHR0..."
FONT_WOFF2 = "d09GRgABAAAAAAZsAA4AAA..."
```

## Configuration

| Parameter         | Default | Description                                              |
|-------------------|---------|----------------------------------------------------------|
| `source`          | —       | Directory or file to encode.                             |
| `output`          | stdout  | Output file for the generated code.                      |
| `format`          | python  | Output format: python, json, js, ts.                     |
| `max_size_kb`     | 512     | Skip files larger than this to prevent huge outputs.     |

## Notes

- This skill reads binary files and writes text files — no network access.
- For very large assets, consider a dedicated asset server rather than
  embedding. The 33% size overhead compounds quickly at multi-MB scale.
- The generated constants are deterministic: running the skill twice on the
  same input always produces the same output.
